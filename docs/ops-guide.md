# 跳海社区 Skill — 运营配合指南

> 本文档面向跳海运营负责人，说明两个需要配合的功能模块：漂流瓶和实时数据同步。
> 本项目由跳海社区成员发起，不是官方项目，但希望得到官方支持。

---

## 一、漂流瓶功能

### 这是什么

一个每日许愿池。用户通过 AI 助手投递匿名/署名留言（如"今晚十点在后海店，穿格子衬衫坐吧台最左边，想找人聊摇滚乐"），每天晚上 8 点统一揭晓当天所有留言。看到的人如果感兴趣，可以去对应的店找人。

本质上是**数字版的愿望箱**——跳海已经在做的事情的线上延伸。

### 跳海需要配合什么

漂流瓶的数据统一存在跳海飞书的多维表格里，和酒单/活动数据放在一起管理。**不需要打酒师做任何额外操作**，但需要运营在飞书里建一个表。

| 事项 | 说明 |
|------|------|
| 改变门店运营流程？ | **不需要** |
| 打酒师需要做什么？ | **不需要** |
| 需要在飞书建一个多维表格？ | **需要**（一次性，5 分钟） |
| 需要日常维护？ | **不需要**（数据由用户通过 AI 写入，自动过期清除） |
| 需要审核内容？ | **不需要**（AI 自动审核，规则已内置） |

#### 飞书多维表格：漂流瓶

| 字段 | 类型 | 说明 |
|------|------|------|
| ID | 文本 | 自动生成，格式 bottle-YYYYMMDD-001 |
| 日期 | 日期 | 自动填充今天 |
| 城市 | 单选 | 北京/上海/深圳/... |
| 门店 | 单选 | 后海店/安定门店/...（可不选，表示该城市所有店可见） |
| 身份 | 单选 | 匿名/留昵称 |
| 昵称 | 文本 | 可选 |
| 留言内容 | 长文本 | 不超过 200 字 |
| 想见面的时间 | 文本 | 如"今晚十点" |
| 状态 | 单选 | sealed（密封）/ revealed（已揭晓）/ expired（已过期） |
| 投递时间 | 日期时间 | 自动记录 |

云函数会在每天 20:00 自动把当天所有 `sealed` 改为 `revealed`，次日自动改为 `expired`。运营无需手动操作。

可选的增强（不是必须）：

- **同步到门店微信群**：让掌群人每天 8 点把当天漂流瓶转发到城市群，或后续对接微信机器人自动转发。
- **反馈违规内容**：如果发现有人利用漂流瓶做不当事情（骚扰、广告），反馈给我们增加过滤规则。

### 安全设计

我们已经内置了以下规则：
- 不允许留电话/微信号/社交账号——线下见面靠描述（穿什么、坐哪）
- 不允许商业推广、色情/骚扰内容
- 每人每天最多 3 个瓶子
- 瓶子有效期当天，次日自动清除

## 二、实时酒单和活动数据

### 这是什么

让用户问 AI "今天后海店有什么酒？" 时能得到真实答案，而不是"建议到店确认"。

### 跳海需要配合什么

这个**需要配合**，但设计目标是**让打酒师/店长每天只多花 2 分钟**。

#### 方案：飞书多维表格

跳海已经全员使用飞书，所有人都会操作多维表格。我们只需要创建两个新的多维表格：

##### 表 1：每日酒单

| 字段 | 类型 | 说明 |
|------|------|------|
| 日期 | 日期 | 自动填充今天 |
| 城市 | 单选 | 下拉选（北京/上海/...） |
| 门店 | 单选 | 下拉选（后海店/安定门店/...） |
| 酒名 | 文本 | 如"夕阳小茉莉" |
| 厂牌 | 文本 | 如"气泡实验室" |
| 风格 | 单选 | IPA/世涛/小麦/酸啤/古斯/其他 |
| 价格 | 数字 | 单杯价格 |
| 备注 | 文本 | 可选，如"今天新上的" |
| 填写人 | 人员 | 自动关联飞书账号 |

##### 表 2：活动日历

| 字段 | 类型 | 说明 |
|------|------|------|
| 日期 | 日期 | 活动日期 |
| 城市 | 单选 | |
| 门店 | 单选 | |
| 活动名称 | 文本 | 如"周五即兴演出之夜" |
| 活动类型 | 单选 | 音乐/市集/展览/读书/播客/运动/其他 |
| 时间 | 文本 | 如"20:00" |
| 描述 | 长文本 | 一两句话说明 |
| 填写人 | 人员 | |

#### 谁来填

| 角色 | 填什么 | 频率 | 耗时 |
|------|-------|------|------|
| **当班打酒师/店长** | 今日酒单 | 每天开店前 | **约 2 分钟**（6-8 个酒名填一下） |
| **城市市场负责人/掌群人** | 本周活动 | 每周一次 | **约 5 分钟** |

#### 怎么让填写无感

1. **飞书多维表格可以设为手机端快捷入口**——打酒师到店后打开飞书，点"今日酒单"，填几个酒名就完了
2. **可以设置模板**——大部分酒头每天变化不大，复制昨天的改一两个就行
3. **可以拍照识别**（后续优化）——拍一张酒头照片，AI 自动识别填表

#### 跳海运营需要做的具体步骤

```
步骤 1：创建飞书多维表格（10 分钟）
  - 在跳海飞书工作区创建三个多维表格：
    ① 每日酒单（按上文字段设置）
    ② 活动日历（按上文字段设置）
    ③ 漂流瓶（按上文字段设置）
  - 把门店列表设为单选选项（三个表共用同一套城市和门店选项）

步骤 2：设置权限（2 分钟）
  - 酒单表：各城市打酒师/店长有编辑权限
  - 活动表：市场负责人有编辑权限
  - 漂流瓶表：不需要任何人手动编辑（由云函数通过 API 读写）

步骤 3：创建飞书开放平台应用（10 分钟）
  - 登录 open.feishu.cn
  - 创建企业自建应用
  - 开通"多维表格"API 权限（读写）
  - 获取 App ID 和 App Secret
  - 把三个表格的 URL 中的 app_token 和 table_id 记下来

步骤 4：把以下信息给到我们
  - App ID
  - App Secret
  - 酒单表的 app_token 和 table_id
  - 活动表的 app_token 和 table_id
  - 漂流瓶表的 app_token 和 table_id

步骤 5：通知打酒师（5 分钟）
  - 在打酒师群发一条通知：
    "从今天开始，到店后顺手在飞书里填一下今天的酒头，
     2 分钟搞定，这样问 AI 助手的人就能知道今天有什么酒了。"
  - 漂流瓶不需要通知任何人——用户自己通过 AI 投递
```

#### 不配合也完全不影响使用

如果跳海暂时不想接入实时数据，**Skill 照常工作**——用户问酒单时会返回通用合作厂牌信息，并提示"以门店实际酒头为准"。我们在技术上已经做好了降级处理。

## 三、打酒师排班数据（Phase 3，暂不需要）

这个需要读取跳海飞书的打酒师排班数据（已经在飞书里管理的抢单系统）。只需要给我们一个只读 API 权限即可。**目前不急，等前两个功能跑通后再考虑。**

---

## 四、总结

| 功能 | 跳海配合度 | 优先级 |
|------|----------|--------|
| 漂流瓶 | ⭐ 建一个飞书表（一次性），之后不需要日常操作 | 最先上线 |
| 实时酒单 | ⭐⭐ 建飞书表 + 打酒师每天填 2 分钟 | 和漂流瓶一起推 |
| 实时活动 | ⭐⭐ 建飞书表 + 市场负责人每周填 5 分钟 | 同上 |
| 打酒师排班 | ⭐⭐⭐ 开放飞书已有排班系统的 API 只读权限 | 后续再做 |

最小启动方案：**一次性创建三个飞书表 + 一个飞书应用，把凭证给到我们。** 漂流瓶立即可用，酒单看打酒师配合节奏逐步填充。
